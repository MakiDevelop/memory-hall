1. **Q1**: `/v1/health` 只看 cached status，非 `ok` 就回 `503`（`src/memory_hall/server/routes/health.py:11-15`）。cache refresh 會真的跑 `embed("healthcheck")`，且任何 sub-check fail 都把整體標成 `degraded`（`src/memory_hall/server/app.py:302-325`）。這些 fail 被吞掉；只有 `_refresh_health_cache()` 整體 raise 才會記 warning（`src/memory_hall/server/app.py:332-340`），所以會出現 silent 503。另 background reindex 每 120s 跑一次（`src/memory_hall/server/app.py:366-374`），文件也承認它會和 health probe 搶 embedder（`docs/benchmarks/results-2026-04-18.md:58-65`）。修法：health 改 cheap probe / last-success cache，至少把 sub-check error 打 log。
2. **Q2**: write path 先存 `pending`，再同步 embed；embed/vector 失敗就回 `202`、entry 留在 `pending`（`src/memory_hall/server/app.py:380-443`）。但 current code **有** auto-reconcile：startup 會起 background reindex（`src/memory_hall/server/app.py:106-114`），每 120s 掃 `sync_status != embedded`（`src/memory_hall/server/app.py:366-485`，`src/memory_hall/storage/sqlite_store.py:260-270`）。我本地縮短 interval 模擬到 `202/pending -> embedded`。所以不是「沒 worker」；真正問題是 write/reindex 都吞 per-entry exception（`src/memory_hall/server/app.py:433-443,497-525`），重試若持續失敗就會無限 pending 且無觀測。修法：記 `last_embed_error/attempted_at` 並 log reindex fail。
3. **Q3**: bearer middleware 只驗 token，不產生 principal（`src/memory_hall/server/app.py:652-676`）；write route 仍用 `get_principal()`（`src/memory_hall/server/routes/memory.py:25-35`）；而 `get_principal()` 只解析 `Authorization: HMAC ...`，否則固定回 `dev-local`（`src/memory_hall/auth/__init__.py:14-20`）。所以 bearer mode 下 `created_by_principal=dev-local` 是現況設計。HMAC roadmap 有，但只在文件：ADR-0007 明說 S2.1 是 destination、先 defer（`docs/adr/0007-minimal-token-auth.md:9-22,35-49`）；而且 `docs/api.md:26-33` 和 ADR 的 header 形狀已不一致。最小落地 PR：repo 內約 150-250 LOC / 5-7 files；若連 6 個 caller 一起切，整體約 250-400 LOC。
4. **附帶觀察**: `src/memory_hall/server/routes/admin.py:10-19` 沒有任何 principal / role gate，但 `docs/design.md:259-265` 寫 admin-only。`/v1/memory/search` 的 semantic failure 也 silent fallback（`src/memory_hall/server/app.py:212-230`）。另外測試對 ambient `MH_API_TOKEN` 不隔離，我本地跑 `pytest -q tests/test_smoke.py tests/test_auth.py` 時有 2 個 auth-related failure。
